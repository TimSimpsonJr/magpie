# Magpie — Structural Map

Fieldwork suite **Magpie** plugin: the FOSS-first FOIA / investigative analysis
toolkit. **Layer 0–1 (laptop-local flagship) is in active development** — most of
the toolkit is still to be built. What exists today is the full structured-data
analysis core (load -> quality-gate -> derive -> served-DB build) plus the
deterministic stats flagship, the 13-point per-source analysis recipe +
cross-source rollup, the Phase 5 authoritative spaCy-NER PII-exposure sweep, and
the Phase 6 Docling document/PDF ingest path (a pure text-layer quality gate ->
DoclingDocument JSON kept internally with page+bbox+charspan provenance), and the
Phase 7 redaction pair -- `redaction-check` (find bad redactions in a PDF as
leads-not-verdicts, dual received/pre-publish mode) + `redact-output` (redact
uninvolved third-party PII to initials for published artifacts; full exhibit kept
local), and the Phase 8 `investigate` verification gate -- `scripts/citation.py`
(a pure char-offset -> text-hash -> block -> page citation anchor over `ingest`'s
preserved DoclingDocument provenance) plus the extraction-verifier / citation-
checker agents -- wired into the skills (`dataset-analyze`, `analysis-recipe`,
`pii-sweep`, `ingest`, `redaction-check`, `redact-output`, `investigate`,
`archive-evidence`) and a bundled read-only `mcp-sqlite` server, and the Phase 9
`archive-evidence` provenance/custody engine -- `scripts/evidence.py`
(SHA-256-on-receipt + an RFC 3161 trusted timestamp from a free TSA + an
append-only hash-chained custody log + a provenance manifest, with a bundled
freeTSA root), and the Phase 10 `setup`/`doctor` onboarding wizard --
`scripts/detect_tier.py` (a pure stdlib capability probe: a golden-testable
capability map in user verbs + a metadata-only / shutil.which / read-only probe
edge, NO new deps) behind an operator `setup` (MAY install) vs a journalist
read-only `doctor` split, with dual operator/journalist onramps and NO Docker;
Track B (entity-network) and heavy infra are later layers and are
NOT yet present.

## Stack

- **Language:** Python 3.12 (pinned 3.12.10 via `mise.toml`). Claude Code plugin (`.claude-plugin/`); hard-depends on the `librarian` plugin.
- **Layer 0–1 deps (laptop-local, no Docker):** pandas 3.0.3, numpy 2.4.6, duckdb 1.5.3, pyarrow 24.0.0; pytest 9.0.3. Phase 3.1 adds openpyxl 3.1.5 (XLSX read/unmerge) + charset-normalizer 3.4.3 (encoding pre-flight). Phase 3.4 adds sqlite-utils 3.39 (build the served SQLite DB) + PyYAML 6.0.2 (test-only: validate the canned-query metadata); mcp-sqlite 0.3.2 runs via `uvx` (not pip-pinned) as a bundled read-only MCP server. Phase 4 (analysis-recipe + rollup) adds NO new deps (pure pandas/numpy reusing the existing modules). **Phase 5 (pii-sweep) adds the FIRST heavy ML dependency: spaCy 3.8.14 + `en_core_web_lg` 3.8.0 (~400 MB wheel, pinned by its reproducible GitHub-release URL), CPU-only (no GPU/CUDA extras), installing against the pinned numpy 2.4.6 WITHOUT touching it (pulls thinc 8.3.13 + blis 1.3.3 CPU BLAS).** pii-sweep uses NER ONLY (PERSON label; tagger/parser/lemmatizer/attribute_ruler disabled). **Phase 6 (ingest) adds the document/PDF stack — HEAVIER than spaCy: docling 2.97.0 (== docling-slim[standard]; pulls a chunking/transformers stack ingest never calls — a tracked footprint trim, prior-art §9) + rapidocr 3.8.1 (the OCR engine Docling 2.97 actually uses — pure-Python + ONNX, NO system binaries; the legacy `rapidocr-onnxruntime` is deliberately NOT pinned) + onnxruntime 1.26.0 (CPU inference) + torch 2.12.0 (pinned to the CPU build so a rebuild can't drift onto CUDA — the layout/TableFormer runtime) + ocrmypdf 17.5.0 (deskew/redo-OCR PREPROCESSING only; needs Tesseract/Ghostscript that pip does NOT manage — detect-and-skipped when absent). Verified at the Phase 6 research gate to install against the pinned numpy 2.4.6 + pandas 3.0.3 WITHOUT moving them; `mise run bootstrap` now also fetches ~1.5–2 GB of wheels + first-run model weights (HuggingFace layout/TableFormer + the RapidOCR ONNX models).** **Phase 7 (redaction-check) adds Free Law `x-ray` 0.3.6 (BSD-2, box-over-live-text detection), which pins `PyMuPDF` (fitz) 1.24.14 as its hard dep -- PyMuPDF is AGPL-3.0, but as a DECLARED pip dep the user installs (never vendored, never bundled-as-binary) it does NOT infect Magpie's MIT license (prior-art §5); PyMuPDF does not depend on numpy/pandas so the pins stay put. The other 7 redaction checks use pikepdf + pdfminer.six (already present via ocrmypdf); only box-over-text needs x-ray, lazily imported.** redact-output reuses the Phase-5 spaCy stack (no new dep). **Phase 9 (archive-evidence) adds `rfc3161-client` 1.0.6 (Apache-2.0) for RFC 3161 trusted timestamping -- its ONLY runtime dep is `cryptography` (already present), it ships a prebuilt cp39-abi3 win_amd64 wheel (no Rust toolchain), and it does NO network (HTTP transport is ours via the already-present `requests`); a bundled freeTSA Root CA (`skills/archive-evidence/references/freetsa_cacert.pem`, valid to 2041, fingerprint-pinned in tests) is the zero-config verify-on-store trust anchor.**
- **Shape:** `scripts/` invoked directly by skills; `pyproject.toml` configures pytest only (not a pip package). **Phase 10 (`setup`/`doctor`) adds NO new deps** -- `scripts/detect_tier.py` probes the toolchain with pure stdlib (`importlib.metadata` + `shutil.which` + bounded `subprocess`), so the dependency surface above is final for Layer 0-1.

```

.mcp.json                     Plugin-root MCP config (auto-discovered). scripts/
  stats.py                    FLAGSHIP deterministic stats module. Generic, decoupled from any corpus loader (inputs are plain sequences / DataFrame + col names). Functions: gini, top_k_share, bottom_half_share, median_by_category, automation_signature, burstiness, presence_rate, category_pct. gini/top_k_share/bottom_half_share drop ALL pandas null sentinels (None/NaN/pd.NA/NaT via pd.isna) before computing, so a nullable Int64 column straight from derive_nets is safe (no nan-poisoning, no NAType float-cast crash). Pure (numpy + pandas), golden-testable.
  load_table.py               Phase 3.1 dirty-data loader (CSV/XLSX -> clean DataFrame + load report). load_table(path, *, encoding, text_columns, empty_null, skiprows, sheet_name, forward_fill_columns, parquet_cache) -> LoadResult(.df, .report). Encoding pre-flight (charset-normalizer) that HONESTLY flags single-byte ambiguity (report.encoding_low_confidence + encoding_alternatives; pin encoding= for byte-exactness), TOKEN-BOUNDARY ID-like TEXT-whitelist (UNIFORM whole-token match for ALL patterns id/case/plate/ssn/dob/zip/zipcode/fips/phone/account/license; leading-zero preservation without coercing real numerics like incident_count/casein or substring-collision names microphone_level/zipper_count/accountability_score; text_columns is the override, report.text_columns lists every coercion), NARROW empty_null (any whitespace-only cell -> NaN consistently across CSV+XLSX via a shared _apply_empty_null sweep; keep_default_na=False so literal N/A/NULL/NaN/None survive as strings — the ***-vs-blank-vs-NULL distinction stays honest), openpyxl unmerge-then-fill for merged XLSX cells + a >2**53 big-int-ID precision warning, skiprows for junk headers, df.ffill() label fill, optional Parquet cache. Pure except file IO; decoupled from stats.py. (pd.read_csv dtype=str is the load; DuckDB is reserved for the Phase 3.3/3.4 query layer.)
  data_quality.py             Phase 3.2 truncation + data-quality GATE (a rigor guardrail: don't publish on incomplete data). check_truncation(rows_or_df, *, ceiling=2**20-1) flags an EXACT-at-ceiling row count (the silent Google-Sheets/CSV 1,048,575 cap) with a request-the-gap message (near-but-below counts NOT auto-flagged). check_date_window(df, date_col, *, requested_start, requested_end) coerces dates (errors=coerce, format=mixed) and flags missing_head/missing_tail when the delivery is NARROWER than the requested window (strict narrower-than; counts n_undated NaT; all-NaT/empty safe). analyze_anomalies(df) emits per-column descriptive LEADS not verdicts (all_null / all_blank / high_null >90% / type_coercion_mismatch for a >80%-numeric-parseable object col with some junk). data_quality_report(df, *, date_col, requested_start, requested_end) aggregates the three. Pure (pandas only), deterministic, no IO/clock/random/network; decoupled from stats.py AND load_table.py (it re-derives the 2**20-1 ceiling rather than importing load_table's copy).
  derive.py                   Phase 3.3 CONFIG-DRIVEN derived-column transform. derive_columns(df, config) -> NEW df (copy; never mutates input) with columns added per config={derivation_name: params}; unknown derivation key -> ValueError (typo can't silently drop a column). Seven separately-importable pure primitives, each driven by config: derive_geo (in/out-of-state/unknown from source_col vs home_value; case/whitespace-insensitive; blank->unknown; output col is `geo`, NEVER `loc` -- pandas .loc collision), derive_reason (reason_text readable-stripped + reason_cat first-matching keyword_map category, default fallback), derive_immigration (bool from a keyword set), derive_nets (pd.to_numeric coerce -> nullable Int64; non-numeric->NA), derive_has_case (presence: any non-blank incl. *** redaction sentinel -> True; blank/NaN/""->False; blank sentinel rejected), derive_base_type ({raw:base} mapping; default label or None=passthrough), derive_temporal_et (parse -> uniform tz-aware UTC -> tz_convert(target_tz) via stdlib zoneinfo -> date_et/hour_et(Int64)/dow_et(Int64 Mon=0); DST-correct; already-tz-aware source converted not re-localized; MIXED naive+aware (or mixed-offset) input handled without raising — aware rows keep their instant, naive rows read as source_tz wall-clock via the _parse_to_utc helper; unparseable->NaT/NA no crash). RIGOR GUARDRAIL: keyword matching is WORD-BOUNDARY (\b-anchored re alternation, longest-first, IGNORECASE) so police/service/justice/notice/iced do NOT trip "ice" (substring trap defeated); a false keyword hit = a false finding. Pure (pandas + stdlib re/zoneinfo), no IO/clock/random/network; decoupled from stats.py/load_table.py/data_quality.py. Phase 4 promoted `_compile_keyword_regex` to a PUBLIC `compile_keyword_regex` + added `keyword_mask(series, keywords)` (the single reusable word-boundary ICE/polICE guardrail) consumed by recipe.py's keyword checks; the private name remains a back-compat alias.
  build_dataset_db.py         Phase 3.4 served-DB builder (the pipeline's "expose" step). build_dataset_db(df, db_path, *, table_name="records", include_columns, text_columns, fts_columns, exclude_columns, pk, replace) -> BuildResult(.db_path, .report). Writes a cleaned/derived DataFrame to the read-only SQLite file mcp-sqlite serves, via sqlite-utils. RIGOR: include_columns is the PREFERRED PII control -- a fail-closed ALLOWLIST that serves ONLY the named columns, so a column the analyst forgets is ABSENT (not exposed); exclude_columns is the fail-open denylist alternative (DROP the named raw-PII columns). Both enforce omission by HARD-DROPPING from the file entirely (mcp-sqlite hidden: is NOT a security boundary -- a hidden table is still SELECTable, so a sensitive column must never enter the file); exclusion is ABSOLUTE and WINS (a column listed in both include & exclude is dropped; exclude also beats text/fts). text_columns get an explicit str schema BEFORE insert (leading-zero IDs survive); every pandas null sentinel (None/NaN/pd.NA incl. nullable Int64/NaT) -> SQL NULL and numpy scalars (np.int64 would else store as a BLOB) -> native python so numeric columns stay numerically queryable; FTS5 (create_triggers) on fts_columns, enabled ONLY on a fresh table (a replace=False append does NOT re-enable FTS). Pure except file IO (closes the connection in a finally so a replace=True rebuild can unlink on Windows); decoupled from the other scripts.
  recipe.py                   Phase 4 -- the 13-point parameterized per-source pass. run_recipe(df, config) runs the configured subset of 13 pure checks (truncation/out_of_state/immigration/pretext/pii/accountability/co_travel/blast_radius/mega_users/operations/ai_moderation/cross_agency/statistical_patterns) over one already-derived frame -> findings {source_id, n_records, checks:{name:result}}. The ORCHESTRATOR layer (unlike its pure siblings it composes stats.py + data_quality.check_truncation + derive.keyword_mask). status in {ok,partial,skipped}; an undefined median/share/gini -> None (never a fake 0) with an observation count; a missing required column -> skipped-with-reason; an unknown check name -> ValueError. Keyword checks (immigration/pretext/co_travel) route through derive's \b-anchored guardrail. Pure/deterministic; native-typed (JSON-able) outputs.
  rollup.py                   Phase 4 cross-source recurrence synthesis. rollup(findings_list) -> {n_sources, recurring_external_actors, aggregate, theses}. Recurrence is EXTERNAL-only (keyed solely off each source's cross_agency.external_actor_counts; no mega-user fallback that would mix entity kinds); cross-source rates are POOLED (Sum num / Sum denom), not a mean of percentages (the unweighted mean is reported but named); a skipped/partial/absent check contributes nothing (no crash, no denominator pollution); undefined pooled rate -> None. Pure (consumes only the findings dicts; no pandas/IO).
  pii_sweep.py                Phase 5 -- the AUTHORITATIVE spaCy-NER + structured-regex PII-EXPOSURE tally over ONE FOIA free-text column. sweep(series, *, person_classifier, patterns, broad_only_names, official_names, collect_local_texts) -> {n_rows, n_nonblank_rows, n_distinct_texts, efficiency_ratio, categories, exposure, [local_texts]}. distinct_texts() runs NER over DISTINCT values (outer whitespace STRIPPED, case PRESERVED -- spaCy is case-sensitive; blanks dropped) then every category is WEIGHTED by the row counts those distinct texts cover (the pilot's ~7x efficiency lesson; efficiency_ratio = nonblank/distinct, None never a fake 0). Each DEFAULT_PII_PATTERN (phone/ssn/email/dob_kw/alien_num=A\d{8,9}/driver_lic[digit-in-body]/race_sex/possible_birthdate) is a SEPARATE category {weighted,distinct} so a low-precision signal can't inflate a high-precision headline; the injectable PersonClassifier splits person_official (named for accountability -- rank/title prefix OR a token-SUBSET match against an official_names lexicon, robust to spaCy span over-extension) from person_unknown_role (a name not matched to an official = a LEAD). exposure.strict is the PUBLISHABLE HEADLINE = the per-row UNION of high-precision PII (officials EXCLUDED; possible_birthdate=bare MM/DD/YYYY & race_sex demoted to BROAD-only leads, BROAD_ONLY_PATTERN_NAMES; override via broad_only_names -- e.g. frozenset() reproduces the pilot figure); exposure.broad = strict + person_unknown_role + the leads. Pure core, spaCy ONLY at the lazy edge: SpacyPersonClassifier._load imports spacy + loads en_core_web_lg (NER-only) on FIRST call, so the distinct/weight/regex/tally logic is golden-testable with a fake classifier and importing the module stays ML-free until sweep runs. The hashed text_id() (stripped-sha256[:16], stable LOCAL join key, collision raises loud) keys the opt-in collect_local_texts redact-output exhibit (off by default; officials-only rows excluded; never published). DECOUPLED from recipe.check_pii (no imports either way; a drift test guards the shared patterns). JSON-able native-typed outputs. **Phase 7 extracted the pure `person_role_in_span(span_text, preceding_token_texts, *, officials, involved, titles) -> "official"|"involved"|"uninvolved"` helper from the inline `_is_official` (behavior-preserving; the SpacyPersonClassifier delegates to it, the existing tests + 2 new boundary-drift guards pin the <=2 title lookback + the len<=2 ent-skip), so Phase-7 `redact_output` shares ONE role rule -- the len<=2 skip stays a per-CONSUMER policy (pii_sweep keeps it for counting; redact_output drops it for redaction).**
  ingest_gate.py              Phase 6 -- the PURE text-layer quality GATE (stdlib only: re/enum/pathlib -- NO docling/PDF/pandas/numpy/spaCy, golden-testable with no model; mirrors pii_sweep's pure-core/edge split). diagnose_page(native_text, *, parse_score, lang, wordlist) -> PageDiagnosis(native_ok|image_only|garbled_text|uncertain_review): ordered first-match rules, each a NAMED MODULE CONSTANT (never a bare branch number) -- (1) <_MIN_CHARS stripped AND no alpha tokens -> image_only (a SHORT-but-present Bates token is NOT image_only, it falls through), (2) <_MIN_TOKENS alpha tokens -> uncertain_review (a sparse form/letterhead -- too little signal to judge a hit-rate), (3) garbled_text REQUIRES CO-OCCURRENCE (wordlist hit-rate <_GARBLED_HIT_RATE AND not char_density_ok) so a low hit-rate ALONE never garbles -- numeric tables / all-caps headers / multi-column run-ons stay native_ok (the false-positive guards), (4) Docling parse_score <=_MIN_PARSE_SCORE contradicts otherwise-acceptable text -> uncertain_review, else native_ok. decide_doc(list[PageDiagnosis]) -> DocDecision(native|ocr_images|force_full_doc_ocr|review): CONSERVATIVE ordered rollup over each diagnosis' SHARE (empty->review; uncertain >=_REVIEW_FRACTION -> review; garbled >=_ESCALATE_FRACTION OR garbled-present-and-(image+garbled)>=fraction -> force_full_doc_ocr; image_only >=fraction -> ocr_images; else native -- a minority of bad pages in a mostly-native brief stays native and is flagged later, NEVER flipping a 200-page doc to full OCR). Signal helpers alphabetic_tokens (lowercased [A-Za-z]+ runs; drops digits/punct, splits hyphens), wordlist_hit_rate (None never a fake 0.0 when no alpha tokens), char_density_ok (two independent NAMED-constant guards: a _MIN_LETTER_RATIO floor catching a wall of pure symbols/digits + a _MAX_NONLETTER_RUN guard catching a long ;;;;/####/16-digit run; a numeric TABLE legitimately has a low letter ratio so the floor is deliberately small). The INJECTABLE wordlist (mirrors pii_sweep's injectable classifier): functions take `wordlist: frozenset|None`; None lazily loads + module-CACHES the bundled common_words.txt (the ONE IO touch-point, read at most once/process), tests inject a tiny synthetic set so the pure suite never reads the file. (str, Enum) enums are JSON-able. Pure/deterministic: no clock/random/network.
  ingest.py                   Phase 6 -- the Docling EDGE (the ONLY docling importer; imports it LAZILY inside `_docling_imports` so importing this module stays cheap, mirroring pii_sweep's lazy spaCy edge). ingest(pdf_path, *, out_dir, lang, deskew, max_num_pages, max_file_size, page_range, document_timeout) -> IngestResult: source_sha256 (streamed 1 MiB-chunk SHA-256 -- artifact identity, design §2.6) -> Pass #1 do_ocr=False (an EXPLICIT ingest profile: do_table_structure OFF for latency, enrichments off, convert() large-doc guards _DEFAULT_MAX_PAGES/_DEFAULT_MAX_FILE_SIZE/page_range, document_timeout=_DEFAULT_DOCUMENT_TIMEOUT 300s wall-clock ceiling, raises_on_error=False) -> UGLY-PDF FAILURE GATE (design §2.6): res.status==ConversionStatus.FAILURE (corrupt/encrypted/unparseable; verified -- pdfium "Data format error" swallowed, res.document exists but res.document.pages=={}) short-circuits to a flagged review IngestResult (trustworthy_for_extraction=False, ocr_engine_used "none", warning naming the FAILING STAGE (pass-1 vs OCR) + status + res.errors summary, one flagged per_page entry per existing page or [], a MINIMAL DoclingDocument JSON still written if a doc object exists else docling_json_path="" + a warning; NEVER dereferences a bad doc for extraction). The failure contract is UNIFORM across BOTH converts via a single `_run_convert(stage, ...)` helper (try/except wraps each so the rare backend that RAISES despite raises_on_error=False routes to the same flagged path + inspects res.status; the failure-result construction is factored into `_build_failure_result`): pass #1 AND the OCR re-convert (ocr_images/force_full_doc_ocr) short-circuit IDENTICALLY -- an OCR pass that times out / FAILUREs / raises returns the flagged review result instead of normalizing/saving/dereferencing a bad doc (Phase 6.2 r2 fix: the OCR re-convert was previously unguarded). PARTIAL_SUCCESS (from EITHER convert) proceeds but warns (_PARTIAL_SUCCESS_WARNING) + flags every page (trustworthy_for_extraction=False) -> pure gate (diagnose each page -> decide_doc) -> APPLY the decision (native/review REUSE pass #1's doc; ocr_images = do_ocr=True force_full_page_ocr=False; force_full_doc_ocr = force_full_page_ocr=True -- both via RapidOcrOptions, ocr_engine_used "rapidocr") -> _normalize_prov_origins forces EVERY item prov bbox to a SINGLE BOTTOMLEFT origin (bbox.to_bottom_left_origin(page_height) is a no-op when already BOTTOMLEFT, safe unconditionally -- the saved-JSON contract requires one coord_origin) -> save_as_json writes the DoclingDocument JSON INTERNALLY (never Markdown -- Markdown drops the bbox/charspan a citation needs) -> Bates post-pass (_BATES_RE word-boundary over iterate_items() text, captured SEPARATELY each with {value, page_no, bbox}; NEVER rewrites text) -> nan-safe degraded flagging. per_page entry = design §5's {page_no, native_chars, hit_rate, diagnosis, ocr_applied, confidence_grade, flagged, flag_reason} PLUS the raw-score debug fields {parse_score, ocr_score} (the re-approved superset; design §5 line updated): hit_rate = wordlist_hit_rate(native_text, load_default_wordlist()) so it matches what the gate scored (None never a fake 0.0 when no alpha tokens); confidence_grade = res.confidence.pages[p].mean_grade (a QualityGrade (str,Enum)) rendered .value, with an absent entry OR UNSPECIFIED -> None (nan-safe, never a fake grade); ocr_applied is PAGE-ACCURATE via _page_ocr_applied(decision, diagnosis) -- True for EVERY page under force_full_doc_ocr, ONLY image_only/garbled_text pages under ocr_images, never under native/review (no doc-wide stamp). The review/`trustworthy_for_extraction` SAFETY CONTRACT: a `review` decision STILL writes the JSON (evidence for human inspection) but sets trustworthy_for_extraction=False and flags every page -- downstream (investigate/citation, Phase 8) MUST refuse auto-extraction; a review doc never silently OCRs. nan IS N/A NOT 0 (design §2.4/§5): res.confidence reports parse_score==nan for an image-only scan and ocr_score==nan for a native page -- _is_real/_score_or_none gate every threshold so a clean native page (ocr_score nan) and a clean scan (parse_score nan) are NOT flagged on a missing-modality score; a page is flagged only when a signal is GENUINELY bad (review/partial, garbled/uncertain/image-only-not-ocr_applied, real ocr_score <_DEGRADED_OCR_SCORE, or an OCRmyPDF op requested-but-skipped). The OCRmyPDF seam is Tesseract-GATED detect/skip/flag only (design §2.5): deskew + `_tesseract_available` (shutil.which probe) absent -> the op is skipped, a warning appended, the pages flagged (never a crash, never a silent skip; the real deskew/redo modes grow in Phase 10 setup-doctor). `sha256_file` + `IngestResult` (a JSON-able-via-asdict dataclass; to_dict(); no docling types leak in) are PURE (no docling). decide_doc/diagnose_page are imported INTO this module's namespace so `scripts.ingest.decide_doc` is the monkeypatch target the review-doc test pins.
  redaction_check.py          Phase 7 (INPUT side) -- find BAD REDACTIONS in a PDF as FLAG-FOR-A-HUMAN LEADS, never an "improper redaction" verdict. `check_redactions(pdf_path, *, mode, vault_roots)` runs 8 layered checks + returns a `RedactionReport`: box_over_text (Free Law x-ray, the ONLY PyMuPDF importer, LAZY -- a missing x-ray / malformed PDF raises `CheckUnavailable` => degrade-don't-crash, NEVER a false "clean"), text_layer (pdfminer; fires only on PAGE-LEVEL co-occurrence with a redaction signal OR an IMAGE-DOMINATED page (max_image_fraction>=0.5) with a substantial hidden text layer -- never a "text exists" alarm), metadata (pikepdf docinfo+XMP), incremental_save (raw bytes: `%%EOF` paired with `startxref` >1), unapplied_redact (`/Subtype /Redact` annot marked-not-applied), embedded_files (attachments + `/AF` + `/FileAttachment`), acroform_values (form `/V`), annotation_text (comment `/Contents`). NEVER-PUBLISH-RAW (design 1.3): every raw leaked string lives ONLY in a finding's `local_evidence`; `detail` carries publishable facts only (counts/field-names/page/bbox+origin); `RedactionReport.publishable_view()` DROPS every local_evidence AND asserts no raw string reached `detail` OR `summary`. Dual `mode`: `received` (leads; safe_to_publish None) / `pre-publish` (FAIL-CLOSED -- safe_to_publish True only if NO `high` finding AND `checks_unavailable` is empty; an un-run check forces False via a pinned `_PREPUBLISH_SEVERITY` map). Always emits a `cannot_catch` honesty footer (glyph-position/pixelation/cross-version/OCG -- no FOSS detector). Coordinate discipline: PAGE-LEVEL co-occurrence only, no cross-engine bbox math (each engine's origin differs). Inlines its own streamed-sha256 (does NOT import scripts.ingest -- decoupled). Offline tier (everything but box_over_text) is PyMuPDF-free on import.
  redact_output.py            Phase 7 (OUTPUT side) -- redact UNINVOLVED third-party PII for PUBLISHED artifacts while the FULL un-redacted exhibit stays LOCAL. POLICY (Tim's call): INVOLVED vs UNINVOLVED, not official vs non-official -- a flagged PERSON name that is NEITHER an official (rank/title prefix OR officials-lexicon subset) NOR an investigator-designated INVOLVED subject (a `keep_names` allowlist) is redacted to INITIALS; officials + involved stay NAMED; SAFE DEFAULT (no keep_names) redacts every non-official flagged name (Simpsonville suspects/POIs/minors). Structured PII -> a TYPED placeholder ([SSN]/[PHONE]/...), masking ONLY the 6 high-precision categories (the 2 broad-only leads race_sex/possible_birthdate are SKIPPED via pii_sweep.BROAD_ONLY_PATTERN_NAMES -- a bare date is an incident date, not auto-PII). Four entry points: `redact_text` (one flagged text), `redact_local_texts` (LOCAL-only text_id->redacted map), `redact_note` (PUBLISH-safe: replaces KNOWN flagged texts in a note, longest-first, narrative untouched), `write_local_exhibit` (the FULL exhibit CSV, the ONLY surface with text_id+raw, VAULT-GUARDED: exhibit_dir + each vault root Path.resolve()'d, RAISES if at/under any vault root). PUBLISH-PATH CONTRACT: only redact_text/redact_note produce published strings, NEITHER carries a text_id or raw text. `_apply_spans` MERGES overlapping spans to their UNION (longest wins; PERSON-before-PII tie; right-to-left) so NO partial-overlap raw prefix/suffix leaks. ## Key Relationships
```

- **`stats.py` is decoupled from any corpus loader.** Every function takes plain values (a sequence of magnitudes, or a `pandas.DataFrame` + column names) — never a Simpsonville-specific loader — so the math is domain-agnostic and reusable across data sources. Docstrings reference the motivating Simpsonville pilot, but nothing imports it.
- **`load_table.py` is the ingest counterpart to `stats.py`, and they are NOT coupled.** The loader turns a dirty CSV/XLSX into a clean DataFrame; `stats.py` consumes a DataFrame. Neither imports the other — a skill wires them (load -> analyze). The loader's UNIFORM whole-token TEXT-whitelist (leading-zero preservation for zip/fips/id/case/... without coercing substring-collision names like microphone_level/zipper_count) and NARROW `empty_null` (literal N/A/NULL survive; any whitespace-only cell -> NaN consistently across both paths) / `***`-vs-blank handling feed the same rigor invariants `stats.presence_rate` enforces downstream. Loader APIs (charset-normalizer `from_bytes`, openpyxl `merged_cells.ranges`+unmerge, pandas `df.ffill()` since `fillna(method=)` is removed in pandas 3, pyarrow Parquet) are pinned to the Phase 3 research gate at `skills/dataset-analyze/references/prior-art.md`.
- **`data_quality.py` is the rigor GATE that sits between ingest and analysis, and it imports neither neighbor.** It takes a DataFrame (or a bare row-count int) and reports truncation / date-window / per-column anomaly LEADS so the analyst doesn't publish on incomplete data. It is intentionally standalone: it RE-DERIVES the `2**20 - 1` truncation ceiling rather than importing `load_table._TRUNCATION_CEILING` (the two are independent on purpose — `load_table` flags the same ceiling as a load-time anomaly note over real file IO, whereas `data_quality` is a pure post-hoc gate over an in-memory frame). Both encode the same documented Google-Sheets/CSV cap; if that constant ever moves it must move in both places. The gate's leads-not-verdicts stance mirrors `load_table`'s honest-ambiguity flagging and `stats`' rigor invariants.
- **`derive.py` is the CONFIG-DRIVEN transform feeding the Phase 4 recipe + `stats.py`, and it imports neither neighbor.** It turns a clean (post-`load_table`, post-`data_quality`) DataFrame into one carrying the derived columns the analysis recipe and stats functions consume — `geo` (→ `stats.category_pct` out-of-state share), `nets` (→ `stats.median_by_category` net-width), `hour_et` (→ `stats.automation_signature`), `has_case` (→ `stats.presence_rate`), `reason_cat`/`is_immigration`/`base_type`. The RULES are entirely in the caller's `config` dict (home state, keyword vocab, type mapping, tz) so the module hardcodes NO Simpsonville values — a different jurisdiction differs only in config. Its WORD-BOUNDARY keyword matching (\b-anchored `re` alternation: police/service/justice do NOT trip "ice") is the SAME rigor stance as `load_table`'s token-boundary TEXT-whitelist and is the categorization half of the keyword-verification guardrail now reused by the Phase 4 recipe via the public `compile_keyword_regex` / `keyword_mask`; the `***`-counts-as-present rule in `has_case` mirrors `load_table`'s `***`-vs-blank handling and `stats.presence_rate`'s presence-≠-value invariant. The `geo`-not-`loc` column name avoids the pandas `.loc` attribute collision. Pure (pandas + stdlib `re`/`zoneinfo`); a skill wires load → quality-gate → derive → analyze.
- **Golden tests ↔ synthetic fixture.** `tests/fixtures/agency_counts_sample.json` is hand-tuned so `gini()` reproduces the pilot's ~0.805 and the top-1%/bottom-50% shares match documented values; `test_stats.py` pins those numbers. The fixture is synthetic — the real corpus is never read in tests.
- **`build_dataset_db.py` is the pipeline's "expose" step and imports none of its neighbors.** It writes the cleaned/derived frame to the read-only SQLite file the bundled `mcp-sqlite` server (`.mcp.json`) serves. Its rigor invariants mirror the upstream ones downstream: PII omission is enforced by HARD-DROPPING columns from the file (mcp-sqlite's `hidden:` is NOT a security boundary -- the agent could still `SELECT` a hidden table). The PREFERRED control is `include_columns`, a fail-closed ALLOWLIST that serves only the named columns so a forgotten column is ABSENT, not exposed; `exclude_columns` is the fail-open denylist alternative (drop named raw-PII columns) and exclusion is ABSOLUTE -- it WINS over include and over text/fts. `text_columns` re-asserts the leading-zero TEXT affinity `load_table` set, the NA-sentinel/numpy-scalar sanitization keeps `derive_nets`' nullable Int64 (pd.NA -> SQL NULL) and numeric columns honest (np.int64 -> native int, not a BLOB), and FTS is enabled ONLY on a fresh table (a `replace=False` append does not re-enable it). The `dataset-analyze` SKILL.md wires the chain (load_table -> data_quality -> derive -> build_dataset_db -> mcp-sqlite query -> stats); `canned_queries.yml` is the row-capped read-only query surface (every query LIMIT-bounded, since mcp-sqlite enforces no cap) and `.mcp.json` launches the pinned server with verified `${CLAUDE_*}` interpolation. The served surface is read-only and PII is omitted by serving an explicit allowlist (or a verified exclude list), not unconditionally.
- **`pii_sweep.py` is the AUTHORITATIVE PII tally; `recipe.check_pii` is the fast PRESENCE indicator — and the two are DECOUPLED.** `pii_sweep` (Phase 5) is the publishable count: spaCy PERSON NER + a broader structured-regex set, run over DISTINCT values then WEIGHTED by row counts (the pilot's ~7× efficiency lesson — weighting a distinct scan equals a naive per-row scan at a fraction of the cost). `recipe.check_pii` (Phase 4) is a fast 4-pattern structured-regex PRESENCE flag inside the per-source pass that explicitly DEFERS the weighted semantic NER to this skill (its `note` says so). They share NO code in either direction: `recipe.py` imports no heavy ML, and `pii_sweep` imports spaCy ONLY lazily inside `SpacyPersonClassifier._load` (first `sweep` call), so importing `pii_sweep` stays ML-free and `recipe.py` never pulls in spaCy. A drift-protection test (`test_pii_sweep.test_overlap_patterns_stay_consistent_with_recipe_check_pii`) is the tripwire that fires if the shared-intent patterns silently diverge (it compares `pii_sweep`'s COMPILED `.pattern` to `recipe._DEFAULT_PII_PATTERNS`' STRINGS; `recipe`'s `a_number` == `pii_sweep`'s `alien_num`, both `A\d{8,9}`). Rigor stance: it SPLITS `person_official` (named for accountability — rank/title prefix OR an `official_names` token-subset match, built from the STRUCTURED searcher column, robust to spaCy span over-extension) from `person_unknown_role` (a name not matched to an official = a LEAD), and EXCLUDES officials from the `strict` (high-precision PII) publishable headline — the per-row UNION of the strict categories, NOT a sum — vs the `broad` leads (`person_unknown_role` + `possible_birthdate` + `race_sex`). Pure-core / spaCy-at-the-edge mirrors `stats`/`derive` purity + `build_dataset_db`'s IO-at-edge: the injectable `PersonClassifier` seam golden-tests the tally with a fake (no 400 MB model). The hashed `text_id` (stripped-sha256, LOCAL-only join key) feeds the opt-in `local_texts` redact seam consumed by `redact-output` (Phase 7) — counts publish, raw texts/`text_id`s NEVER cross a published path (design §7); the `pii-sweep` SKILL.md wires `dataset-analyze` prep → `sweep` → Librarian (aggregate only) → local exhibit. The `\b`-anchors are this module's keyword-rigor analogue (it does NO keyword categorization, so `derive.keyword_mask` is N/A — considered, not overlooked).
- **`ingest_gate.py` is the pure GATE; `ingest.py` is the lazy-Docling EDGE — the same pure-core / engine-at-the-edge split as `pii_sweep`.** `ingest_gate` imports ONLY stdlib (`re`/`enum`/`pathlib`), so the per-page `diagnose_page` → conservative `decide_doc` rollup is golden-testable with no model and the wordlist is INJECTABLE (mirrors `pii_sweep`'s injectable classifier; `None` lazily caches the bundled `common_words.txt`). `ingest.py` is the ONLY docling importer and imports it LAZILY (inside `_docling_imports`, like `pii_sweep`'s lazy spaCy edge), so importing the module stays ML-free — empirically: importing `scripts.ingest` pulls in neither docling, pandas, nor numpy. The **`DoclingDocument` JSON is kept INTERNAL (never Markdown)** because Markdown drops the `{page_no, bbox, charspan}` provenance triple that is the seam to the **Phase-8 `investigate` citation anchor** (`_normalize_prov_origins` first forces every prov bbox to a SINGLE `coord_origin`, defusing the research-gate coordinate-origin trap); a `review` doc still writes that JSON but sets `trustworthy_for_extraction=false` so downstream auto-extraction refuses it. `ingest` is the **document/PDF** path — the structured-data **CSV/XLSX** path is `dataset-analyze` (`load_table` + `data_quality`, factored in Phase 3), which `ingest` deliberately does NOT duplicate. And `ingest` shares **NO code** with the Track-A modules (`stats`/`load_table`/`derive`/`recipe`/`pii_sweep`): the only contract between the two paths is the eventual Librarian findings output and the §7 citation schema. The verified Docling/RapidOCR/OCRmyPDF facts (the coordinate-origin trap, the `nan`-is-N/A confidence semantics, the CPU-latency budget, the Tesseract system-binary gate) are pinned to the Phase-6 research gate at `skills/ingest/references/prior-art.md`.
- **`redaction_check.py` (input side) + `redact_output.py` (output side) are the Phase-7 publish-critical pair — decoupled from everything, pure-core / engine-at-the-edge like `pii_sweep`/`ingest`.** `redaction_check` FINDS bad redactions in a PDF (8 layered leads-not-verdicts checks); its safety invariant is NEVER-PUBLISH-RAW — every raw leaked string lives ONLY in a finding's `local_evidence`, and `publishable_view()` drops it + asserts no raw string reached `detail`/`summary` (route only `publishable_view()` to Librarian, keep `to_dict()` LOCAL). Its `box_over_text` is the ONLY PyMuPDF importer (LAZY -> a missing x-ray/malformed PDF raises `CheckUnavailable` => degrade-don't-crash, and pre-publish FAILS CLOSED on any unavailable check); the offline tier (the other 7 checks: pikepdf + pdfminer + raw bytes) is PyMuPDF-free on import. It re-implements its streamed-sha256 INLINE (does NOT import `scripts.ingest`) and does NO cross-engine bbox math (page-level co-occurrence only). `ingest` (Phase 6) PRESERVED live-text-under-redaction-boxes specifically so `box_over_text` can reason about it (the preserve-don't-fix seam). `redact_output` REDACTS uninvolved third-party PII for PUBLISHED artifacts (the INVOLVED-vs-UNINVOLVED policy via a `keep_names` allowlist — officials + involved kept, uninvolved -> initials, safe default redacts non-officials; structured PII -> typed placeholders for the 6 high-precision categories only) while the FULL un-redacted exhibit is written to a VAULT-GUARDED local path (`Path.resolve()` containment). Its publish-path contract: only `redact_text`/`redact_note` produce published strings, neither carries a `text_id` or raw text; `_apply_spans` MERGES overlapping spans to their union so no partial-overlap raw prefix/suffix leaks. It CONSUMES `pii_sweep`'s opt-in `local_texts` (the `text_id` join) and imports ONLY `pii_sweep`'s PURE helpers (`person_role_in_span`/`_norm_name_tokens`/`DEFAULT_PII_PATTERNS`/`text_id`) — importing it is ML-free (lazy `SpacyPersonSpans`); the shared `person_role_in_span` keeps the official/involved role rule in ONE place (the `len<=2` ent-skip is a per-CONSUMER policy: pii_sweep keeps it for counting, redact_output drops it for redaction since a missed name is a leak). The verified x-ray/pikepdf/pdfminer facts + the AGPL(PyMuPDF)-does-not-infect-MIT call + the involved-vs-uninvolved policy are pinned to the Phase-7 research gate at `skills/redaction-check/references/prior-art.md`.
- **`citation.py` is the pure Phase-8 anchor ENGINE; the `investigate` skill is the ORCHESTRATOR — the same pure-core / engine-at-the-edge split as the rest of the suite.** `citation.py` resolves over a plain DoclingDocument JSON dict (no docling/spaCy import), so the fallback-chain resolver is golden-testable with hand-built dict fixtures (`conftest_citation`). Its only seams: it CONSUMES `ingest`'s preserved `{page_no, bbox, charspan}` provenance and REFUSES a non-trustworthy (`review`/`PARTIAL_SUCCESS`) doc — the contract `ingest`'s SKILL already documents — and `public_anchor()` feeds the Librarian publish path. The verification PIPELINE lives in the `investigate` SKILL, not the engine: extract → the structurally-separate `extraction-verifier` (semantic, advisory) + `citation-checker` (mechanical, drives `resolve_anchor`/`is_clean_citation`) → the mandatory solo human gate → `redact_note(claim_text, local_texts)` (the Phase-7 publish-safe path) + `public_anchor`. The skill is the ONLY place that touches `derive.keyword_mask` (the ICE/polICE guardrail) and `redact_output`; `citation.py` stays free of both. The never-publish-raw discipline (raw quote / claim / context + verifier/checker reasoning are LOCAL-only) mirrors Phase-7 `redaction_check`'s `local_evidence` / `publishable_view`. The anchor format is a magpie invention validated EARLY on the real Greenville RFP (16 relocated, 0 false exacts across a re-ingest), pinned at `skills/investigate/references/prior-art.md`.
- **`evidence.py` is the Phase-9 provenance/custody ENGINE; the `archive-evidence` skill is the ORCHESTRATOR — the same pure-core / engine-at-the-edge split as the rest of the suite.** The pure core (streamed `sha256_file`, the hash-chained custody log, the manifest assembly + the `_full_verification` choke-point normalization, the mtime alarm, the `archive_evidence` orchestration) is stdlib-only + clock-INJECTED, so it golden-tests with a `FakeTimestamper` (mirroring `pii_sweep`'s injectable classifier + `citation.py`'s injected timestamp). The lazy `Rfc3161Timestamper` edge imports `rfc3161-client` + `requests` ONLY inside its method, so importing the module stays network-free (like `ingest`'s lazy docling). Its safety invariants: hash-ON-RECEIPT before any processing (the provenance anchor); the TOCTOU imprint guard (the token's messageImprint MUST equal the published receipt sha256, else `imprint_mismatch`, never a silent `verified` — the same never-fake-it rigor as `ingest`'s nan-is-N/A and `redaction_check`'s leads-not-verdicts); FAIL-CLOSED verification (a non-freeTSA TSA with no configured root → `unverified`, never opportunistic trust); the manifest is the ATOMIC final commit point (temp + `os.replace`) and the `archived` custody event lands ONLY after it (so the append-only log never claims an archive that did not commit; any post-commit append failure surfaces as the explicit, recoverable `CustodyAppendError`); a TSA failure soft-degrades (a specific `unavailable`/`unverified` reason) while a LOCAL write failure hard-fails. The never-overstate discipline (the custody log is tamper-EVIDENT not tamper-PROOF; a timestamp attests a hash existed at a time, NOT authorship/source/unaltered-since; revocation is NOT checked) mirrors the suite's honest-limits stance (design 7). It imports NO Librarian — the SKILL routes the provenance note while the raw token + local paths stay LOCAL (the local-vs-published split, like `redaction_check`'s `publishable_view` and `citation.py`'s `public_anchor`). The verified RFC-3161 facts + the live-smoke validation (a freeTSA round-trip + an `openssl ts -verify` cross-check) are pinned at `skills/archive-evidence/references/prior-art.md`.
- **Private corpus is gitignored, by design.** The real Flock/Simpsonville FOIA data (names, DOBs, plate context) lives OUTSIDE the repo; `.gitignore` carries an explicit belt-and-suspenders block (`corpus/private/`, `*-Audit.csv`, `**/*[Ss]impsonville*[Aa]udit*`, exhibit files) so PII can never be committed. `corpus/public/` is reserved for the future redistributable sample.
- **Hard dependency on Librarian.** `plugin.json` `dependencies: ["librarian"]` (and a manifest test asserting it) wires Magpie's findings output to the shared Librarian notes layer; this is auto-installed with Magpie. Soft couplings to Research / Prose Craft are design-level only, not yet wired.
- **Plan → code traceability.** `docs/plans/2026-06-03-magpie-layer-0-1.md` is the executable plan and names `scripts/stats.py` as its Phase 2 deliverable; the design doc is its cited source of truth. Per-phase plans/design docs land alongside it (e.g. `2026-06-04-magpie-phase5-pii-sweep*.md`, `2026-06-04-magpie-phase6-ingest*.md`, `2026-06-04-magpie-phase7-redaction*.md`). The plan drives what lands next (Phase 6 Docling/OCR ingest + Phase 7 redaction-check/x-ray + redact-output + the Phase-8 `investigate` citation anchor over `ingest`'s preserved provenance + the Phase-9 `archive-evidence` provenance/custody engine now present; then setup/doctor and the Task-11 validation against the real corpus), so the tree will keep growing under `scripts/`, `skills/`, and `agents/` as phases complete.
- **Dev environment is mise-managed.** `mise.toml` pins Python 3.12.10 and binds the project `.venv` (which carries the deps). Run tests via `mise run test`, rebuild the venv via `mise run bootstrap`, run ad-hoc Python via `mise exec -- python ...`. The Claude Code PowerShell tool runs `-NoProfile`, so mise's bare-command *activation* can't auto-load there -- use `mise run`/`mise exec`, and never bare `python` (it resolves to the global interpreter, whose dep versions may differ from the venv -- e.g. global pandas 3.0.1 vs the venv's pinned 3.0.3). The always-works fallback is the explicit `.venv\Scripts\python.exe` path. Tests assume `pythonpath ["."]` from `pyproject.toml`.
- **`detect_tier.py` is the Phase-10 onboarding probe; `setup` + `doctor` are the two skills over it — the same pure-core / IO-at-edge split as the rest of the suite, plus a hard write/read asymmetry.** The capability-MAP logic (`build_capability_map` / `summarize` / `render_text`) is PURE and golden-tested from injected probe dicts (mocked presence/absence, like `pii_sweep`'s injectable classifier and `ingest_gate`'s injectable wordlist); the `check_*` probes are the only IO edge (stdlib `importlib.metadata` / `shutil.which` / bounded `subprocess`, NO new deps, no heavy imports — probing torch/docling/spacy via `metadata.version` never loads them, so `doctor` stays fast). The map is keyed by USER VERBS (analyze datasets / ingest native PDFs / OCR preprocessing for scans / PII scan / redaction QA / citation verify / evidence timestamp), each gating the skill whose dist/binary deps it checks; capabilities are INDEPENDENT (a missing OCR binary makes ONLY that one capability UNAVAILABLE, never cascading) and the headline is a SUBORDINATE binary READY/NOT-READY + a document-workflows rollup, never a linear 1/2/3 tier. The setup/doctor ASYMMETRY is the whole point: `setup` (operator, once) MAY install (`mise run bootstrap` + INSTRUCT for the system binaries pip can't manage); `doctor` (journalist, anytime) is strictly READ-ONLY (never installs, never executes uvx, never starts the mcp-sqlite server — the Codex risk fix). `README.md` + `docs/OPERATOR_GUIDE.md` + `docs/JOURNALIST_START.md` are the dual onramp; NO Docker anywhere (Layer 2, out of scope), pinned by a guard test over all three surfaces. Honest limit: it reports presence + version, not correctness.
